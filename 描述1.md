# HarmonyOS 页面跳转示例工程文件描述

## 项目概述

本项目是一个基于 HarmonyOS ArkUI 框架的示例应用，展示了 ArkTS 页面与 H5 页面之间的相互跳转功能。通过 Web 组件加载 H5 页面，并实现 H5 页面与 ArkTS 页面之间的参数传递、导航控制等功能。项目名称为 "PageRedirection"，包名为 `com.example.pageredirection`。

**核心功能**：
- 在 ArkTS 页面中嵌入 Web 组件加载 H5 商品列表页面
- H5 页面点击商品跳转到商品详情 H5 页面
- H5 页面通过 JavaScript 调用 ArkTS 方法跳转到 ArkTS 订单确认页面
- 支持 Web 组件的导航控制（前进、后退、首页）
- 实现沉浸式状态栏和导航栏适配

**技术栈**：
- HarmonyOS SDK 5.0.5 (API 17)
- ArkTS 语言
- ArkUI 声明式开发范式
- Web 组件（@kit.ArkWeb）
- 响应式 UI 布局

### 3. 源代码文件

#### CommonConstants.ets
定义应用中的常量值：
- 字体家族：HarmonyHeiTi
- 百分比常量：100%、50%
- 吐司持续时间：2000ms
- 间距：12vp

#### Logger.ets
日志工具类，基于 `@kit.PerformanceAnalysisKit` 的 hilog 模块实现，提供 debug、info、warn、error 等级别的日志输出。

#### ProductModel.ets
数据模型接口：
- `ProductModel`：商品模型，包含 id、name、img、price、sku 字段
- `ArkTSFunModel`：ArkTS 功能模型，定义 `jumpOrderConfirm` 方法供 H5 调用

#### EntryAbility.ets
应用入口 Ability，继承自 `UIAbility`，主要功能：
- 生命周期方法：onCreate、onDestroy、onWindowStageCreate、onWindowStageDestroy、onForeground、onBackground
- 在 onWindowStageCreate 中加载首页 `pages/IndexPage`

#### IndexPage.ets
主页面，包含 Web 组件和导航控制：
- `@Entry` 和 `@Component` 装饰器
- 状态变量：`webCanBack`、`webCanForward`、`controller`、`statusBarHeight`、`sliderBarHeight`
- `arkTSObj`：暴露给 H5 的 JavaScript 代理对象，包含 `jumpOrderConfirm` 方法
- 生命周期方法：`aboutToAppear`、`onPageShow`、`onPageHide`、`onBackPress`
- Web 组件配置：加载 `product_list.html`，设置 JavaScript 代理
- 导航按钮：后退、前进、首页按钮，根据 Web 历史状态启用/禁用
- 沉浸式状态栏适配

#### OrderConfirmPage.ets
订单确认页面：
- 从路由参数获取 `statusBarHeight`、`sliderBarHeight` 和商品详情数据
- 展示商品信息、用户地址、配送方式、发票信息等
- 包含返回按钮和确认订单按钮
- 使用自定义样式扩展 `titleStyle`

### 5. H5 页面文件

#### product_list.html / product_list.css / product_list.js
- 商品列表页面，展示 6 个商品
- 点击商品跳转到 `product_detail.html?index=商品索引`
- 使用 CSS 实现列表布局

#### product_detail.html / product_detail.css / product_detail.js
- 商品详情页面，展示商品图片轮播、价格、详情信息

## 功能说明

### 1. ArkTS 与 H5 页面交互
- **ArkTS → H5**：通过 Web 组件加载 H5 页面，使用 `javaScriptProxy` 将 ArkTS 对象暴露给 H5
- **H5 → ArkTS**：H5 通过 `arkTSFunObj.jumpOrderConfirm()` 调用 ArkTS 方法，传递商品详情数据

### 2. 页面导航
- **Web 导航**：支持在 H5 页面间前进、后退、返回首页
- **ArkTS 导航**：使用 `Router` 进行页面跳转，参数传递
- **系统返回键处理**：在 `onBackPress` 中处理 Web 页面后退

### 3. 状态栏适配
- 获取系统避让区域（状态栏和导航栏高度）
- 动态调整页面 padding 实现沉浸式效果
- 根据 Web 页面状态改变状态栏颜色

### 4. 数据传递
- **H5 到 ArkTS**：通过 JavaScript 代理传递 JSON 字符串
- **ArkTS 页面间**：通过路由参数传递 `statusBarHeight`、`sliderBarHeight` 和商品详情数据

### 5. 错误处理
- 使用 `try-catch` 捕获异常
- 通过 Logger 工具记录错误日志
- 处理窗口获取、Web 控制器操作等可能失败的操作

## 使用流程

1. **启动应用**：显示商品列表 H5 页面
2. **浏览商品**：点击商品跳转到商品详情 H5 页面
3. **页面导航**：
   - 点击底部返回按钮：Web 页面后退
   - 点击前进按钮：Web 页面前进（如果可用）
   - 点击首页按钮：返回商品列表页面
4. **下单流程**：在商品详情页点击"立即购买"，跳转到 ArkTS 订单确认页面
5. **订单确认**：查看订单信息，点击"确认订单"显示提示

1. **混合开发**：结合 ArkTS 原生 UI 和 H5 页面
2. **双向通信**：ArkTS 与 H5 通过 JavaScript 代理交互

4. **完整导航**：支持 Web 页面历史导航和原生页面跳转
