# 页面重定向应用 - 新增功能项描述

## 项目概述
本项目是一个基于HarmonyOS ArkTS开发的电商应用演示，展示了从Web页面到ArkTS页面的完整跳转流程。相较于最初只能跳转一次的简单版本，当前版本实现了完整的电商购物流程和丰富的交互功能。

## 核心新增功能

### 1. Web与ArkTS深度集成
- **Web组件嵌入**：在ArkTS页面中嵌入Web组件，加载本地HTML商品列表页面
- **JavaScript与ArkTS双向通信**：通过`javaScriptProxy`实现Web页面JavaScript调用ArkTS方法
- **ArkTS方法暴露**：将`jumpOrderConfirm`方法暴露给Web页面调用

### 2. 完整的电商购物流程
- **商品列表展示**：Web页面显示商品列表，支持点击查看详情
- **商品详情页面**：独立的商品详情HTML页面，展示商品图片轮播和详细信息
- **订单确认页面**：ArkTS实现的订单确认页面，显示收货地址、商品信息、配送方式等
- **购买流程**：商品列表 → 商品详情 → 订单确认的完整流程

### 3. 丰富的用户交互
- **对话框系统**：
  - 购买确认对话框（"彳亍" / "有的是"选项）
  - 二次确认对话框（"......." / "你玩我是吧？"选项）
- **应用终止功能**：用户选择"你玩我是吧？"时，应用会优雅终止
- **页面返回控制**：支持Web页面前进/后退导航，ArkTS页面返回按钮

### 4. 系统级功能集成
- **沉浸式状态栏**：获取系统状态栏和导航栏高度，实现沉浸式布局
- **动态状态栏颜色**：根据Web组件状态动态调整状态栏颜色（亮色/暗色模式）
- **系统栏属性管理**：通过`window.Window`API管理系统栏显示和颜色

### 5. 页面导航管理
- **页面栈管理**：使用`router.clear()`清空页面栈，`router.pushUrl()`跳转页面
- **参数传递**：通过路由参数传递状态栏高度、商品详情等数据
- **返回控制**：`onBackPress()`方法处理物理返回键逻辑

### 6. 错误处理与日志系统
- **全面的错误处理**：所有API调用都有try-catch错误处理
- **日志记录**：使用统一的Logger工具记录操作日志和错误信息
- **用户体验保障**：错误时显示友好的提示，避免应用崩溃

### 7. 资源管理与本地化
- **多语言支持**：通过`string.json`管理所有文本资源
- **资源引用**：使用`$r()`方法引用资源文件（图片、字符串、尺寸、颜色等）
- **样式统一**：通过`CommonConstants`管理公共样式和尺寸

## 技术架构亮点

### ArkTS组件架构
```typescript
@Entry
@Component
struct IndexPage {
  @State controller: webview.WebviewController = new webview.WebviewController();
  @State statusBarHeight: number = 0;
  @State sliderBarHeight: number = 0;
  // ...
}
```

### Web与ArkTS通信机制
```typescript
.javaScriptProxy({
  object: this.arkTSObj,
  name: 'arkTSFunObj',
  methodList: ['jumpOrderConfirm'],
  controller: this.controller
})
```

### 路由参数传递
```typescript
// 跳转时传递参数
this.getUIContext().getRouter().pushUrl({
  url: 'pages/OrderConfirmPage',
  params: { statusBarHeight: this.statusBarHeight, sliderBarHeight: this.sliderBarHeight, detailStr }
});

// 接收参数
statusBarHeight: number = (this.getUIContext().getRouter().getParams() as Record<string, number>)['statusBarHeight'];
```
